LU分解及求逆

LU分解中，L是下三角矩阵，U是上三角矩阵，并且L的对角线为1。计算L和U的过程中需要交替计算，具体是先计算U的一行，然后计算L的一列，再计算U的一行。在python中使用行操作将不需要对每一个单元进行操作。另外，值得注意的是，求取L和U的过程看成L、U对矩阵的行操作和列操作将有助于理解和代码的编写。
**注意注意：**这里必须要提醒一下，python数组和列表的=表示的仅仅是引用，并没有拷贝对象的意思，因此，当需要一个中间量tmp存储一个数组中的部分元素时，最好使用np.copy()而不是直接=，因为随着原数组的改变，tmp中的值会跟着改变。
a=[1,2,3]
c=a
a.append(4)
c=[1,2,3,4]
注意，copy和deepcopy的区别，当使用copy时表示的是浅拷贝，deepcopy顾名思义就是深拷贝。使用浅拷贝时，列表或者数组里面的子列表或者数值不会被拷贝，例如a=[1,2,[3,4]]中的[3,4]并没有拷贝，仅仅是引用，而深拷贝则会全部拷贝。

SVD分解

  关于SVD分解，这里有一个很好的视频教学，网址。

看完视频后，我自己对SVD的理解主要分为两个部分：
1.svd分解得到的三个矩阵，U,sigma,V在几何形状上作用的含义不同。U和V的作用可以看做相同，即将图形旋转。而sigma矩阵的作用是将图像做拉伸。
2.svd分解可以用于图像压缩。需要注意的是，sigma矩阵中对角线的元素总是大于等于0的。看到“sigma矩阵中对角线的元素总是大于等于0的”这句话的一瞬间可能很难理解，但是换一个角度就比较容易理解了。可以理解为sigma中的对角线元素是一个系数，将U和V.T的列向量和行向量相乘所得的矩阵相加即得到原来的矩阵。当系数很小的时候，完全可以将其忽略掉，这就是数据压缩的原理。这也引出一个问题就是怎么计算U和V矩阵。计算的推导运用到了逆向推导的方法，即先假定原矩阵A是可以分解的。然后通过AA.T和A.TA，可以得出U为AA.T的特征向量的组合，而V既是A.TA的特征向量的组合。还需要注意的是，特征向量可以取正值也可以取负值，那需要什么样的原则确定呢？我暂时有一个想法是：U[0][i]V[0][i]>0,即通过不断的相加而趋近于原矩阵。如果小于0，我们需要将其U或者V的第i列乘一个-1。
